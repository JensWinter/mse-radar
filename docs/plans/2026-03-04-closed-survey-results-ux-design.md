# Closed Survey Results UX Design Plan

## 1. Benchmark Context

- Show interpretation bands near scores so numbers are immediately meaningful:
  - `1-2`: Beginning
  - `3-4`: Developing
  - `5-7`: Mature

## 2. Delta vs Previous Run

- Add a **Change since last run** value per capability (for example, `+0.8` or `-0.4`).
- Include up/down indicators for quick scanning.

## 3. Key Takeaways Block

- Auto-highlight:
  - Top 2 strengths.
  - Top 2 biggest improvement opportunities.
  - Largest disagreement signal (where variance is high).

## 4. Variability Indicator

- Keep the aggregated score.
- Also show spread/confidence (for example, `6 responses, high alignment` vs `mixed opinions`).

## 5. Action-Focused Guidance

- Convert each capability card into:
  - Why this matters.
  - Next 1-2 actions.
  - Optional lightweight follow-up tracking fields (owner and target date).

## 6. Better Information Hierarchy

- In detail cards, prioritize:
  - Team score (largest).
  - Your score (secondary).
  - Response count, trend, and guidance below.

## 7. Share/Export

- Add quick actions:
  - Copy summary to clipboard.
  - Export PDF/CSV for team retrospectives.

## 8. Empty/Low-Data Clarity

- If only 1-2 responses are available, show a clear reliability warning so teams do not over-interpret the results.
