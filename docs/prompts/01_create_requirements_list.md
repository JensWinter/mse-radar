We initiate a new web application. First read docs/project_vision.md to find out about the project's vision.

Let's start by planning the application and brainstorm some ideas for it. Create a file docs/requirements.md. List possible functional requirements (title and description) and prioritize them.

Some of the main functions will be:
- provide survey functionality to assess software engineering skills of a dev team based on the DORA capabilities
  - survey can be answered by all team members
  - survey can be run multiple times during a project lifecycle
  - team leads can manage (create/open/close) surveys
- visualize a team's current capabilities based on survey results
- provide selective guidance on how to develop certain areas of capabilities (using DORA research program)
- LLM/chat bot functionality

Divide the requirements into sections according to MoSCoW prioritization categories.

Requirements should contain a number in the format 0001, 0002, etc.
Use the following format for each requirement:

### 0001: [Requirement Title]
**Priority:** [Must have / Should have / Could have / Won't have]
**Description:** [A brief description of the requirement]

The requirements (including numbering) should be sorted by priority (Must have first, then Should have, etc.) and within each priority category sorted by requirement number.
